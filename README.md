## 📋 О проекте

Компания занимается розничной продажей товаров медицинского назначения (матрасы, подушки и прочие изделия). Более 95% покупателей предпочитают оплачивать покупку только после получения товара. Часть клиентов отказываются от приобретения, что приводит к убыткам.

**Проектная задача** — построить модель, которая по данным о заказе в день продажи предсказывает вероятность выкупа. Это позволит компании своевременно реагировать на заказы с низким шансом выкупа, оптимизировать процессы и снизить убытки.

### Бизнес-ценность

Зная вероятность выкупа, компания может:

| Действие | Эффект |
|----------|--------|
| 🔄 Усилить отработку заказа | Снижение отказов на 15-20% |
| 🚚 Изменить тариф доставки | Оптимизация логистических затрат |
| 📦 Изменить логистическую схему | Ускорение обработки |
| 💳 Изменить условия оплаты | Снижение финансовых рисков |

## 📂 Форматы данных

### Входные данные (CSV)
Должны содержать колонки, описанные в `config.py` (DANGER_COLUMNS будут удалены автоматически).

Обязательные колонки:
- `lead_id` — идентификатор заказа
- `buyout_flag` — целевая переменная (1 — выкуп, 0 — отказ) — нужна только для обучения
- `lead_Состав заказа` — текстовое описание для извлечения артикулов и цен
- Остальные колонки — автоматически обрабатываются

### Выходные данные

**predictions.csv** (после обучения):
| Колонка | Описание |
|---------|----------|
| `lead_id` | ID заказа |
| `y_true` | Истинная метка (1/0) |
| `y_pred_raw` | Сырая вероятность от CatBoost |
| `y_pred_calibrated` | Откалиброванная вероятность |
| `y_pred_optimal_threshold` | Класс при пороге 0.42 |
| `y_pred_70_percentile` | Класс при 70-м персентиле |

**new_predictions.csv** (на новых данных):
| Колонка | Описание |
|---------|----------|
| `lead_id` | ID заказа (если есть) |
| `prediction_proba` | Откалиброванная вероятность |
| `prediction_class` | Класс при пороге 0.42 |

## 🎯 Бизнес-интерпретация порогов

| Порог | Precision | Recall | Когда использовать |
|-------|-----------|--------|-------------------|
| **0.42** | 82.3% | 99.6% | Основной сценарий: минимизация пропущенных выкупов |
| **0.78** | 93.1% | 34.1% | Когда важнее точность (ограниченные ресурсы на обработку) |

**Рекомендация:** Использовать порог **0.42** как основной. При пороге 0.42:
- 99.6% выкупных заказов будут правильно идентифицированы
- Только 0.4% выкупных заказов попадут в "серую зону"
- 82.3% заказов, помеченных как "высокий риск", действительно требуют внимания
